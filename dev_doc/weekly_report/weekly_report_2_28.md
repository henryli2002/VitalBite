# 本周工作汇报（Weekly Report）

## 1. 已完成 (Completed)
- **多模型 Router 意图识别深度评测**：
  - 完成了 OpenAI (4o-mini)、Gemini、Claude (3 Haiku) 在 Intent Router 模块的横向效果对比评测。
  - **OpenAI 4o-mini** 表现最优，意图识别准确率达到 **95.8%**（相比 Baseline 的 25.0% 有显著提升），单次请求时间、Token 消耗与综合成本均处于理想范围。
  - **超参数 Grid Search 性能分析与性价比权衡**：
    - **OpenAI 4o-mini & Claude 3 Haiku**: 在低 Temperature (`0.0 - 0.2`) 与低 Top-P (`0.5`) 的设定下，对处理 **Prompt Injection（指令攻击）** 的鲁棒性明显更好，有效降低了幻觉和被绕过指令的概率，因此其最高准确率均达到了 **95.8%**。
    - **Gemini (Flash 2.5 Lite)**: 整体准确率在 90.8% ~ 94.2% 之间波动（峰值 94.2%）。其与前两者的核心差异主要来源于面对复杂 Prompt 攻击时的防御表现。**但从成本效益（Cost-efficiency）来看，Gemini 的价格优势显著**：官方 API 收费仅约为 OpenAI 4o-mini 的 **2/3**，更是 Claude 3 Haiku 的 **1/10**。在大规模并发或对成本敏感的业务场景中，具有极高的性价比。
- **系统工程层面的 Retry 与延迟 (Latency) 分析**：
  - 在核心路由和澄清节点实现了最大重试机制（`for attempt in range(3)`）并配合指数退避/休眠（`sleep(1)`），有效防止了因瞬间并发过高或网络抖动导致的 API 调用直接崩溃，极大增强了系统容错率。
  - 从横向评测的时间表现来看，整体评测耗时非常接近（Claude 总计 280s, OpenAI 291s, Gemini 297s），换算后**最新平均单次请求推理延迟 (Latency) 稳定在约 2.3秒 ~ 2.5秒**，证明在实际业务场景中处于稳定且可接受的范围内。
- **Prompt Injection 与系统安全防御机制实现**：
  - 在 Router 层实现了前置的安全防御策略，通过正则匹配（英文/中文指令注入、Prompt 截断符识别、Meta-Control 覆盖规则等）对用户恶意输入进行阻断。
  - 成功将存在潜在攻击风险（Prompt Attack）、无食用价值或有害安全（如腐败食物图片）的输入精准路由至 `clarification` 进行隔离与澄清，确保下游推荐/识别节点的系统安全。

## 3. 下周计划 (Next Week's Plan)
- **图像识别能力服务化拆分（Image Analysis Agent）**：
  - 计划将**“图片识别（Image Recognition）”单独拆分出去作为一个独立的 Agent 服务**。
  - **核心价值**：
    1. **提高多图理解能力**：更专注地处理和理解多图输入上下文。
    2. **加强图片攻击防御能力**：通过前置独立 Agent，在图像特征层面进行深度清洗和防御，增强安全性。
    3. **辅助推荐偏好分析**：在 Recommendation 模块中，可以通过该独立识图 Agent 分析用户上传的图片，精准辨别用户的视觉偏好（口味、菜系等），从而提供更个性化的餐厅与食物推荐。
- **完善全局 Log 记录机制**：
  - 计划在后续工作中增加全面的数据日志记录（Logging），覆盖系统状态流转、用户异常输入捕捉、API 请求延迟以及重试信息，从而支持更好的链路追踪和线上 Debug 排查。
