# 微调模型评估报告与技术选型分析

本文档旨在提供关于项目中微调模型（`finetuned`）的详细评估报告，并阐述在进行食物营养识别任务时，为何选择端到端的图像回归模型，而非更传统的目标检测（Object Detection）模型。

---

## Part 1: 微调模型核心评估

### 1.1 模型概述

- **模型架构**: 采用轻量级的 **`MobileNetV3-Small`** 作为骨干网络。该模型的顶层被替换为一个回归头，可以直接输出5个连续的数值。详细的训练流程和模型定义可见于 [`eval/train_model.py`](eval/train_model.py:1)。
- **核心任务**: 这是一个**端到端（End-to-End）**的回归任务。模型接收一张完整的食物图片作为输入，直接预测出整份餐食的**5个核心宏观营养指标**：
    1.  总质量 (克)
    2.  总卡路里 (大卡)
    3.  总脂肪 (克)
    4.  总碳水 (克)
    5.  总蛋白质 (克)

### 1.2 核心评估结果

评估结果来源于对30个样本的严格测试，并将微调模型与其他三种方法（`graph`流程、`direct`零样本LLM、`fewshot`少样本LLM）进行了对比。详细数据见 [`eval/results/final_analysis_report.txt`](eval/results/final_analysis_report.txt:1)。

#### 1.2.1 准确性对比 (wMAPE 指标，越低越好)

| 营养指标 | **`finetuned` (微调模型)** | `graph` (旧流程) | `direct` (LLM零样本) | `fewshot` (LLM少样本) |
| :--- | :--- | :--- | :--- | :--- |
| **总质量** | **21.5%** | 62.5% | 54.0% | 72.1% |
| **卡路里** | **21.7%** | 135.4% | 79.3% | 51.0% |
| **总脂肪** | **22.2%** | 117.9% | 56.2% | 72.5% |
| **总碳水** | **22.8%** | 93.7% | 39.4% | 64.2% |
| **总蛋白质**| **30.8%** | 102.3% | 52.0% | 71.4% |

**结论**: 微调模型在所有关键指标上均取得了**压倒性的胜利**。与原有的 `graph` 流程相比，其**误差率降低了4到5倍**，证明了其卓越的预测准确性。

#### 1.2.2 性能（速度）对比

| 方法 | 平均执行时间 (每张图片) |
| :--- | :--- |
| **`finetuned` (微调模型)** | **0.1 秒** |
| `direct` (LLM零样本) | 3.2 秒 |
| `fewshot` (LLM少样本) | 4.4 秒 |
| `graph` (旧流程) | 13.0 秒 |

**结论**: 微调模型的速度极快，**比旧流程快130倍**，比LLM方法快30-40倍，这对于实现低延迟的用户体验和降低服务成本至关重要。

## Part 2: 技术选型分析：为何不使用目标检测模型？

目标检测（Object Detection，如YOLO、Faster R-CNN等模型）是计算机视觉的常见技术，它可以在图片中定位并识别出不同的物体（例如，识别出盘子里的“苹果”、“米饭”、“鸡翅”并用方框标出）。然而，对于本项目的**核心目标——估算整餐的总营养**，该方法存在以下几个关键的局限性，因此我们选择了端到端的回归模型。

### 2.1 目标检测模型的局限性

1.  **任务不匹配，流程复杂**:
    *   目标检测的核心任务是**“识别”和“定位”**，而不是“量化”。即使模型成功识别出所有食物，我们仍然需要一个额外的、极其复杂的后续步骤来**估算每个食物方框内的体积和重量**，然后再去查询数据库、最后再汇总。这个流程漫长且每一步都会引入新的误差。
    *   **正确答案**: `图片 -> 总营养`
    *   **目标检测流程**: `图片 -> N个食物方框 -> N个食物类别 -> N个重量估算 -> N次数据库查询 -> 汇总`

2.  **标注成本极高**:
    *   训练一个优秀的目标检测模型，需要**成千上万张图片**，并且每张图片都需要**人工精确地为每一种食物绘制边界框（Bounding Box）**。这是一个极其耗时且昂贵的数据标注过程。
    *   相比之下，我们当前采用的回归模型仅需要**图片级别的标签**（整张图片对应的总营养值），数据获取成本要低得多。

3.  **难以处理混合与非固态食物**:
    *   目标检测非常擅长处理轮廓清晰的物体，但很难处理像**“牛肉炖土豆”、“蔬菜沙拉”、“燕麦粥”**这样的混合物或半流体。你无法为“炖菜的汤汁”画一个有意义的方框。
    *   而回归模型通过观察整张图片的外观、颜色和纹理，可以学习到这些混合菜肴的整体营养特征。

4.  **无法解决体积估算难题**:
    *   一个2D的方框无法准确地代表食物的3D体积和重量。一盘堆起来的薯条和一个平铺的薯条，方框可能一样大，但重量天差地别。如何从2D边界框准确推断重量，本身就是一个巨大的技术挑战。

### 2.2 端到端回归模型的优势

我们选择的 `MobileNetV3` 回归模型恰好解决了以上所有问题：

-   **流程简单直接**: 模型一步到位，直接输出最终答案，避免了多阶段流程的误差累积。
-   **数据标注高效**: 仅需图片级别的营养标签，大大降低了数据门槛。
-   **学习整体特征**: 模型将整张图片视为一个整体，能够更好地理解混合食物和食材搭配的整体营养状况。
-   **速度极快**: 单一的轻量级模型保证了极低的计算延迟。

**最终建议**: 对于**估算一餐总营养**这一特定目标，端到端的图像回归模型是比目标检测模型更直接、更高效、也更务实的技术方案。它完美地平衡了准确性、性能和开发成本。
